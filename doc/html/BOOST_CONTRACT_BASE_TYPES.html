Macro BOOST_CONTRACT_BASE_TYPES

BOOST_CONTRACT_BASE_TYPES — Used to program a typedef listing the bases of a derived class.

Synopsis

// In header: <boost/contract/base_types.hpp>

BOOST_CONTRACT_BASE_TYPES(...)

Description

In order to support subcontracting, a derived class that specifies contracts for one or more overriding public function must declare a typedef named BOOST_CONTRACT_BASE_TYPEDEF using this macro. This typedef must be public unless boost::contract::access is used.

This is a variadic macro. On compilers that do not support variadic macros, the base classes typedef can be programmed manually without using this macro (see Advanced Topics).

See Also:

Tutorial

Parameters:

...

Comma separated list of base classes. Each listed base must explicitly list its access specifier public, protected, or private, and virtual if present (this not always required in C++ instead). There is an intrinsic limit around 20 on the maximum number of supported bases (because of similar limits for some Boost libraries like Boost.MPL internally used by this library).
